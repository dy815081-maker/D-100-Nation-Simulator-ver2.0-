# D-100 Nation Simulator — 망한 나라 살리기
"파산 위기의 가상국가 — 당신이 대통령입니다"
100일 안에 나라를 구하거나, 역사에 오명을 남기거나.

---

## 1. 프로그램 소개

이 프로젝트는 **파산 위기의 가상국가**를 스토리텔링 배경으로,  
실제 사회적 요소(정책·경제·파벌·외교)를 **Python OOP 체계**로 설계한  
정책 선택형 웹 시뮬레이션 게임입니다.

플레이어는 100일간 대통령이 되어 복잡하게 얽힌 이해관계를 조정하며  
나라를 정상화(또는 붕괴)로 이끕니다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| 트레이드오프 | 모든 정책은 좋아지는 분야와 나빠지는 분야가 동시에 존재 |
| 랜덤 이벤트 | 매일 30% 확률로 돌발 사건 발생 — 계획이 흔들린다 |
| 파벌 반응 | 5개 파벌이 정책마다 개별적으로 반응 |
| 외교 보호막 | 외교 정책 선택 시 10일간 국제 이벤트 피해 50% 감소 |
| 쿨타임 시스템 | 정책 10회마다 1시간 쿨타임 발생 |
| 코인 상점 | 정책 실행으로 코인 적립 → 아이템 구매 |
| 17가지 엔딩 | 플레이어의 선택에 따라 완전히 다른 결말 |

## 3. 이 세계관(프로그램)을 실행하고 즐기는 법

### 1) 게임 세계관 이해하기

이 게임의 배경은 **파산 직전의 가상 국가**입니다.  
플레이어는 이 나라의 대통령이 되어, 제한된 100일 동안 국가를 다시 안정시키는 것을 목표로 합니다.

국가는 현재 재정 악화, 경제 침체, 물가 불안, 복지 부족, 치안 문제, 환경 문제, 낮은 지지율 등 여러 위기를 동시에 겪고 있습니다.  
따라서 플레이어는 단순히 하나의 문제만 해결하는 것이 아니라, 여러 지표의 균형을 맞추며 국가를 운영해야 합니다.

게임 속에서 플레이어가 선택하는 정책은 모두 장점과 단점을 함께 가지고 있습니다.  
예를 들어 복지를 확대하면 국민 만족도와 지지율은 올라갈 수 있지만, 국가 재정은 악화될 수 있습니다.  
반대로 긴축 재정을 선택하면 재정은 좋아질 수 있지만, 복지나 지지율이 떨어질 수 있습니다.

이처럼 이 프로그램은 대통령의 입장에서 정책 선택의 책임과 사회적 균형의 중요성을 체험할 수 있도록 설계된 시뮬레이션 게임입니다.

### 2) 게임을 즐기는 방법

플레이어는 매일 하나의 정책을 선택하며 게임을 진행합니다.
정책을 선택하면 국가의 여러 지표가 변화합니다.

관리해야 하는 주요 지표는 다음과 같습니다.

+ 재정(/br)
+ 경제(/br)
+ 물가안정(/br)
+ 치안(/br)
+ 교육(/br)
+ 복지(/br)
+ 환경(/br)
+ 지지율(/br)

**각 지표는 0부터 100 사이의 값을 가지며, 특정 지표가 너무 낮아지면 국가 운영이 어려워집니다.**

게임은 단순히 높은 점수를 얻는 것이 아니라, 여러 지표가 무너지지 않도록 균형 있게 관리하는 것이 중요합니다.
정책을 선택할 때는 “지금 당장 좋은 선택”과 “장기적으로 나라를 살리는 선택” 사이에서 고민해야 합니다.

### 3) 랜덤 이벤트와 위기 관리

게임을 진행하다 보면 매일 일정 확률로 랜덤 이벤트가 발생할 수 있습니다.
예상치 못한 경제 위기, 국제 문제, 사회적 갈등 등이 발생하며, 이로 인해 국가 지표가 갑자기 변할 수 있습니다.

따라서 플레이어는 항상 여유 있는 지표 관리를 해야 합니다.
특히 재정과 지지율이 너무 낮은 상태에서 큰 위기가 발생하면 게임 오버로 이어질 수 있습니다.

외교 정책을 선택하면 일정 기간 국제 이벤트로 인한 피해를 줄일 수 있으므로, 위기 상황에 대비하는 전략으로 사용할 수 있습니다.

### 4) 게임 오버 조건

플레이어가 국가 운영에 실패하면 게임이 종료될 수 있습니다.

대표적인 게임 오버 조건은 다음과 같습니다.

재정이 0 이하가 되면 국가 파산
지지율이 0 이하가 되면 대규모 폭동
종합지수가 너무 낮은 상태가 3일 연속 지속되면 정권 붕괴

따라서 하나의 지표만 높이는 것보다 전체적인 안정성을 유지하는 것이 중요합니다.

### 5) 엔딩을 보는 방법

100일 동안 국가를 유지하면 최종 지표에 따라 다양한 엔딩이 결정됩니다.

예를 들어 경제를 크게 성장시키면 경제 호황 엔딩을 볼 수 있고, 복지를 중심으로 운영하면 복지국가 엔딩을 볼 수 있습니다.
환경 지표를 높게 유지하면 녹색 전환 엔딩이 나타날 수 있으며, 외교 정책을 잘 활용하면 외교 대박 엔딩도 가능합니다.

반대로 재정 관리에 실패하거나 지지율이 낮아지면 국가 파산, 대규모 폭동, 정권 붕괴와 같은 부정적인 엔딩을 맞이할 수 있습니다.

즉, 이 게임은 플레이어의 선택에 따라 나라의 미래가 달라지는 다중 엔딩 시뮬레이션 게임입니다.

### 6) 더 재미있게 플레이하는 팁

한 지표만 올리기보다 전체 지표의 균형을 맞추는 것이 중요합니다.
재정이 낮을 때는 복지 확대나 대규모 투자를 신중하게 선택해야 합니다.
지지율이 낮아지면 폭동 위험이 커지므로 국민 반응도 함께 고려해야 합니다.
외교 정책은 국제 이벤트 피해를 줄일 수 있으므로 전략적으로 사용하면 좋습니다.
여러 번 플레이하면서 서로 다른 정책 조합과 엔딩을 확인해보는 재미가 있습니다.


## 4. 게임 규칙

### 핵심 지표 (8개)

| 지표 | 설명 |
|------|------|
|  재정 (Fiscal) | 국가 예산 / 부채 관리 |
|  경제 (Economy) | 경제 성장률과 고용 |
|  물가안정 (PriceStability) | 인플레이션 / 생활 물가 |
|  치안 (Safety) | 치안과 사회 안전망 |
|  교육 (Education) | 교육 수준과 인적 자본 |
|  복지 (Welfare) | 사회 복지 체감도 |
|  환경 (Environment) | 환경 보호 수준 |
|  지지율 (Approval) | 대통령 지지율 |

모든 지표는 **0 ~ 100** 범위로 제한됩니다.

### 난이도 배율

| 난이도 | 배율 | 특징 |
|--------|------|------|
|  Easy | × 0.8 | 정책 효과 완화 |
|  Normal | × 1.0 | 기본 |
|  Hard | × 1.25 | 정책 효과 증폭 |

### 게임 오버 조건

Python 예외로 구현된 3가지 게임 오버:

```
GameOverException      → 종합지수 평균 < 35 가 3일 연속
NationBankruptException → 재정(Fiscal) 0 이하
RevoltException         → 지지율(Approval) 0 이하
```

### 하루 진행 순서

```
정책 선택 → 지표 변화 → 파벌 반응 → 이벤트 판정(30%) → 게임 오버 판정
```

## 5. OOP 설계 구조

### 클래스 상속 다이어그램

```
Entity (부모)
├── Government          국가 정부 — 8개 지표 보유, 정책/이벤트 효과 적용
├── CitizenGroup        파벌 집단 — react_to_policy() 다형성
├── Company             기업 집단 — invest() 경제 변화 반응
└── ShopItem (부모)     상점 아이템 — buy() 다형성
    ├── CooldownSkipItem
    ├── CrisisPotionItem
    ├── BuffPotionItem
    ├── AdvisorItem
    ├── PopularityItem
    ├── EconomyBoostItem
    ├── SecurityItem
    └── HonorItem

Policy (dataclass)      정책 객체
Event  (dataclass)      이벤트 객체
```

### 주요 매직 메소드

| 클래스 | `__init__` | `__str__` | `__repr__` | `__len__` |
|--------|-----------|-----------|------------|-----------|
| Entity | o | o | o | - |
| Government | o | o | o | o (경과 일수) |
| CitizenGroup | o | o | o | o (만족도) |
| Company | o | o | - | - |
| ShopItem | o | o | - | - |

### 다형성 예시

```python
# ShopItem.buy() — 8개 하위 클래스에서 각각 다르게 동작
class CrisisPotionItem(ShopItem):
    def buy(self, gov):
        sorted_stats = sorted(gov.stats.items(), key=lambda x: x[1])[:2]
        for k, _ in sorted_stats:
            gov.stats[k] = clamp(gov.stats[k] + 20)
        return f"위기 해소! {names} +20 회복"

class PopularityItem(ShopItem):
    def buy(self, gov):
        gov.stats["Approval"] = clamp(gov.stats["Approval"] + 25)
        return "여론 캠페인 성공! 지지율 +25"
```

## 8. 엔딩 목록 (17가지)

| 엔딩 | 조건 요약 |
|------|----------|
| 국가 정상화 | 종합지수 ≥ 62, 재정 ≥ 52, 지지율 ≥ 52 |
| 외교 대박 | 외교 정책 사용 + 경제 ≥ 60, 지지율 ≥ 58 |
| 경제 호황 | 경제 ≥ 75, 재정 ≥ 62 |
| 복지국가 | 복지 ≥ 65, 지지율 ≥ 55, 재정 < 45 |
| 기업 천국 | 경제 ≥ 70, 복지 < 38 |
| 녹색 전환 | 환경 ≥ 70, 종합지수 ≥ 48 |
| 교육 혁명 | 교육 ≥ 72, 경제 ≥ 60 |
| 치안 국가 | 치안 ≥ 72, 복지 < 35 |
| 물가 지옥 | 경제 ≥ 50, 물가안정 < 25 |
| 막판 대역전 | 후반 15일 평균 - 전반 15일 평균 ≥ 15 |
| 외교 참사 | 국제 이벤트 3회 이상 + 외교 미사용 |
| 안정적 통치 | 종합지수 ≥ 55 |
| 기술관료 정부 | 교육 ≥ 70, 경제 ≥ 65, 복지 < 35 |
| 무난한 타협 | 종합지수 ≥ 40 |
| 국가 파산 | 재정 ≤ 0, 경제 < 30 |
| 대규모 폭동 | 지지율 ≤ 0, 치안 < 25 |
| 정권 붕괴 | 위 조건 모두 미해당 |
