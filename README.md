# D-100 Nation Simulator — 망한 나라 살리기

> “파산 위기의 가상국가 — 당신이 대통령입니다.”  
> 100일 안에 나라를 구하거나, 역사에 오명을 남기거나.

---

## 1. 프로그램 소개

이 프로젝트는 **파산 직전의 가상국가**를 배경으로 한 **Python 콘솔 기반 정책 선택형 시뮬레이션 게임**입니다.

플레이어는 위기에 빠진 국가의 대통령이 되어 100일 동안 정책을 선택하고, 국가의 핵심 지표를 관리하며 나라를 정상화시키는 것을 목표로 합니다.

이 게임은 단순히 숫자를 올리는 게임이 아니라, 현실 사회에서 나타나는 정책 선택의 딜레마를 객체지향 프로그래밍(OOP)으로 모델링한 프로젝트입니다.  
복지를 늘리면 국민 만족도는 올라가지만 재정이 악화될 수 있고, 긴축 재정을 선택하면 예산은 회복되지만 지지율과 복지가 떨어질 수 있습니다.  
즉, 모든 선택에는 장점과 단점이 함께 존재하도록 설계했습니다.

---

## 2. 설치 및 실행 방법

### 필수 준비

- Python 3.8이상
- VS Code 또는 Python 실행이 가능한 터미널

### 실행 방법

압축을 푼 뒤, `main.py`가 있는 폴더에서 터미널을 열고 다음 명령어를 입력합니다.

```bash
python main.py
```

Windows에서 위 명령어가 되지 않으면 다음 명령어를 사용할 수 있습니다.

```bash
py main.py
```

프로그램이 실행되면 콘솔 화면에 게임 설명, 현재 국가 지표, 선택 가능한 정책 목록이 출력됩니다.  
플레이어는 숫자를 입력해 정책을 선택하며 게임을 진행합니다.

## 3. 이 세계관(프로그램)을 실행하고 즐기는 법

### 1) 게임 세계관 이해하기

이 게임의 배경은 **파산 직전의 가상 국가**입니다.  
플레이어는 이 나라의 대통령이 되어 제한된 100일 동안 국가를 다시 안정시키는 것을 목표로 합니다.

국가는 재정 악화, 경제 침체, 물가 불안, 복지 부족, 치안 문제, 환경 문제, 낮은 지지율 등 여러 위기를 동시에 겪고 있습니다.  
따라서 플레이어는 하나의 문제만 해결하는 것이 아니라, 여러 지표의 균형을 맞추며 국가를 운영해야 합니다.

게임 속 정책은 모두 장점과 단점을 함께 가지고 있습니다.  
예를 들어 복지 확대는 복지와 지지율을 올릴 수 있지만 재정을 악화시킬 수 있고, 긴축 재정은 재정을 회복시키지만 복지와 지지율을 낮출 수 있습니다.

이처럼 이 프로그램은 대통령의 입장에서 정책 선택의 책임과 사회적 균형의 중요성을 체험할 수 있도록 설계된 시뮬레이션 게임입니다.

### 2) 게임을 즐기는 방법

플레이어는 매일 하나의 정책을 선택하며 게임을 진행합니다.  
정책을 선택하면 국가의 여러 지표가 변화합니다.

관리해야 하는 주요 지표는 다음과 같습니다.

- 재정
- 경제
- 물가안정
- 치안
- 교육
- 복지
- 환경
- 지지율

각 지표는 0부터 100 사이의 값을 가지며, 특정 지표가 너무 낮아지면 국가 운영이 어려워집니다.

게임은 단순히 높은 점수를 얻는 것이 아니라, 여러 지표가 무너지지 않도록 균형 있게 관리하는 것이 중요합니다.  
정책을 선택할 때는 “지금 당장 좋은 선택”과 “장기적으로 나라를 살리는 선택” 사이에서 고민해야 합니다.

### 3) 랜덤 이벤트와 위기 관리

게임을 진행하다 보면 일정 확률로 랜덤 이벤트가 발생할 수 있습니다.  
예상치 못한 경제 위기, 국제 문제, 사회적 갈등 등이 발생하며, 이로 인해 국가 지표가 갑자기 변할 수 있습니다.

따라서 플레이어는 항상 여유 있는 지표 관리를 해야 합니다.  
특히 재정과 지지율이 너무 낮은 상태에서 큰 위기가 발생하면 게임 오버로 이어질 수 있습니다.

외교 정책을 선택하면 일정 기간 국제 이벤트로 인한 피해를 줄일 수 있으므로, 위기 상황에 대비하는 전략으로 사용할 수 있습니다.

### 4) 게임 오버 조건

플레이어가 국가 운영에 실패하면 게임이 종료될 수 있습니다.

대표적인 게임 오버 조건은 다음과 같습니다.

- 재정이 0 이하가 되면 국가 파산
- 지지율이 0 이하가 되면 대규모 폭동
- 종합지수가 너무 낮은 상태가 3일 연속 지속되면 정권 붕괴

따라서 하나의 지표만 높이는 것보다 전체적인 안정성을 유지하는 것이 중요합니다.

### 5) 엔딩을 보는 방법

100일 동안 국가를 유지하면 최종 지표에 따라 다양한 엔딩이 결정됩니다.

예를 들어 경제를 크게 성장시키면 경제 호황 엔딩을 볼 수 있고, 복지를 중심으로 운영하면 복지국가 엔딩을 볼 수 있습니다.  
환경 지표를 높게 유지하면 녹색 전환 엔딩이 나타날 수 있으며, 외교 정책을 잘 활용하면 외교 대박 엔딩도 가능합니다.

반대로 재정 관리에 실패하거나 지지율이 낮아지면 국가 파산, 대규모 폭동, 정권 붕괴와 같은 부정적인 엔딩을 맞이할 수 있습니다.

즉, 이 게임은 플레이어의 선택에 따라 나라의 미래가 달라지는 다중 엔딩 시뮬레이션 게임입니다.

### 6) 더 재미있게 플레이하는 팁

- 한 지표만 올리기보다 전체 지표의 균형을 맞추는 것이 중요합니다.
- 재정이 낮을 때는 복지 확대나 대규모 투자를 신중하게 선택해야 합니다.
- 지지율이 낮아지면 폭동 위험이 커지므로 국민 반응도 함께 고려해야 합니다.
- 외교 정책은 국제 이벤트 피해를 줄일 수 있으므로 전략적으로 사용하면 좋습니다.
- 여러 번 플레이하면서 서로 다른 정책 조합과 엔딩을 확인해보는 재미가 있습니다.

## 4. 게임 규칙

### 핵심 지표 8개

| 지표 | 설명 |
|---|---|
| 재정 (Fiscal) | 국가 예산과 부채 관리 |
| 경제 (Economy) | 경제 성장률과 고용 상황 |
| 물가안정 (PriceStability) | 인플레이션과 생활 물가 안정 |
| 치안 (Safety) | 사회 안전과 범죄 대응 |
| 교육 (Education) | 교육 수준과 미래 인재 양성 |
| 복지 (Welfare) | 국민 생활 안정과 사회 안전망 |
| 환경 (Environment) | 환경 보호와 지속 가능성 |
| 지지율 (Approval) | 대통령에 대한 국민 신뢰도 |

모든 지표는 **0부터 100 사이**의 범위로 제한됩니다.

### 하루 진행 순서

```text
정책 선택 → 지표 변화 → 파벌 반응 → 랜덤 이벤트 판정 → 게임 오버 판정 → 다음 날 진행
```

### 난이도 배율

| 난이도 | 배율 | 특징 |
|---|---:|---|
| Easy | × 0.8 | 정책 효과가 완화되어 비교적 쉬움 |
| Normal | × 1.0 | 기본 난이도 |
| Hard | × 1.25 | 정책 효과가 크게 적용되어 더 어려움 |

### 게임 오버 조건

게임 오버는 사용자 정의 예외 클래스로 처리됩니다.

```text
GameOverException        → 종합지수 평균이 낮은 상태가 3일 연속 지속될 때
NationBankruptException → 재정(Fiscal)이 0 이하가 될 때
RevoltException          → 지지율(Approval)이 0 이하가 될 때
```

## 5. OOP 설계 구조

이 프로젝트는 국가 운영 시스템을 여러 객체로 나누어 설계했습니다.

### 주요 클래스

| 클래스 | 역할 |
|---|---|
| Entity | 공통 속성과 출력 형식을 제공하는 부모 클래스 |
| Government | 국가의 8개 지표, 날짜, 정책 적용, 게임 오버 판정 관리 |
| CitizenGroup | 노동자, 기업, 청년층 등 사회 집단의 정책 반응 관리 |
| Company | 기업 집단의 투자와 경제 반응 관리 |
| Policy | 정책 이름, 카테고리, 효과를 저장하는 정책 객체 |
| Event | 랜덤 이벤트 이름, 설명, 효과를 저장하는 이벤트 객체 |
| ShopItem | 상점 아이템의 공통 부모 클래스 |
| 하위 ShopItem 클래스들 | 각 아이템마다 서로 다른 buy() 동작 구현 |

### 클래스 상호작용

```text
Policy 선택
   ↓
Government가 국가 지표 변화 적용
   ↓
CitizenGroup이 정책에 반응
   ↓
Company가 경제 상황에 따라 투자/반응
   ↓
Event가 발생하면 Government 지표 추가 변화
   ↓
예외 조건이 발생하면 사용자 정의 예외로 게임 종료
```

## 6. OOP 원칙 적용

### 1) 상속

`Entity` 클래스를 부모 클래스로 두고, `Government`, `CitizenGroup`, `Company`, `ShopItem` 등이 공통 구조를 물려받도록 설계했습니다.  
이를 통해 객체마다 공통적으로 필요한 이름, 설명, 문자열 출력 기능을 재사용할 수 있습니다.

### 2) 다형성

상점 아이템은 모두 `ShopItem`을 상속하지만, 각 아이템의 `buy()` 메소드는 서로 다르게 동작합니다.

예를 들어 어떤 아이템은 위기 지표를 회복시키고, 어떤 아이템은 지지율을 올리며, 어떤 아이템은 경제를 강화합니다.  
같은 `buy()`라는 이름의 메소드라도 객체 종류에 따라 다른 결과를 내기 때문에 다형성이 적용되었습니다.

### 다형성 예시

```python
class ShopItem:
    def buy(self, gov):
        pass

class CrisisPotionItem(ShopItem):
    def buy(self, gov):
        # 가장 낮은 지표를 회복
        return "위기 지표를 회복했습니다."

class PopularityItem(ShopItem):
    def buy(self, gov):
        # 지지율 상승
        return "지지율이 상승했습니다."
```

## 7. 매직 메소드 활용

이 프로젝트는 `__init__` 외에도 여러 매직 메소드를 활용했습니다.

| 매직 메소드 | 활용 내용 |
|---|---|
| `__str__` | 객체를 사람이 읽기 쉬운 문자열로 출력 |
| `__repr__` | 개발자가 객체 상태를 확인하기 쉽게 표현 |
| `__len__` | Government에서는 경과 일수, CitizenGroup에서는 만족도처럼 객체의 상태를 숫자로 표현 |

예를 들어 `print(government)`를 실행하면 단순한 메모리 주소가 아니라 현재 국가 상태를 이해하기 쉬운 문장으로 출력할 수 있습니다.  
또한 `len(government)`를 사용해 현재까지 진행된 날짜를 확인할 수 있도록 설계했습니다.

## 8. 사용자 정의 예외 처리

국가 운영 실패 상황은 일반적인 오류가 아니라 게임 세계관에 맞는 사건으로 처리했습니다.  
따라서 직접 예외 클래스를 만들어 게임 오버 상황을 구분했습니다.

| 예외 클래스 | 발생 상황 |
|---|---|
| GameOverException | 종합지수가 낮은 상태가 3일 연속 지속될 때 |
| NationBankruptException | 재정이 0 이하가 되어 국가가 파산할 때 |
| RevoltException | 지지율이 0 이하가 되어 대규모 폭동이 발생할 때 |

이 예외들은 게임 진행 중 조건을 만족하면 발생하며, 프로그램은 해당 예외를 처리해 게임 오버 메시지와 결과를 출력합니다.
